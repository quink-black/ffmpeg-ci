# 性能优化项目执行规则

> 适用于所有性能优化、硬件加速、算法改进类项目

---

## Rule 0: 永不编造数据

- 每个性能数字（%、ms、speedup）必须注明：测试工具、配置参数、测试机器
- 没有实测数据时回答"needs measurement"，绝不估算

---

## Rule 1: 测量先行——编码前必须完成

**任何优化代码编写前，必须完成并呈现：**

1. **识别热点**: `perf record` (cycles) → 目标函数的CPU占比 → **T_save（预期节省）**
2. **估算开销**: 最小开销估计（数据传输、同步、API调用）→ **T_cost（新增成本）**
3. **计算净收益**: **Net gain = T_save − T_cost**
4. **Go/No-Go判断**: 若 Net gain < 5%，报告"not viable"并建议中止

**绝不跳过此步骤。**

---

## Rule 2: 指令数 ≠ 实际时间

- `callgrind` 统计的是指令数，不是周期数
- SIMD密集型代码中，指令数与周期数的差距可达 **2–3×**
- **速度up估算只能用 `perf record` (cycles)**，callgrind仅用于定位热点

---

## Rule 3: 架构可行性验证——强制检查清单

**编码前必须回答以下问题：**

### 数据来源验证
- [ ] 优化目标的数据从哪里来？
- [ ] 数据在何时生成？何时被消费？
- [ ] 数据存储在哪里？能否直接访问？

### 时序验证
- [ ] 数据在优化点是否已可用？
- [ ] 是否存在异步/同步依赖？
- [ ] 数据生命周期是否与优化假设一致？

### 架构约束检查
- [ ] 是否存在中间buffer/转换层？
- [ ] 临时数据结构的存在是否有必然原因？
- [ ] 画出数据流图：Source → [Transform A] → [Buffer B] → Destination

**未通过检查清单 → 禁止编码。**

---

## Rule 4: Go/No-Go 是硬门槛

- 未通过Go/No-Go → 默认 **No-Go**
- 要覆盖决策：必须提供量化的收益上界和成本下界，并获得确认
- **绝不用定性推理（"理论上应该可行"）绕过量化否决**

---

## Rule 5: 失败时先挑战假设

**优化失败时的正确做法：**

| 错误做法 | 正确做法 |
|---------|---------|
| 立即修复表面症状 | 重新审视根本假设 |
| 连续尝试多种修复 | 验证假设是否成立 |
| 假设数据在X位置 | 确认数据是否真在X位置 |

**流程：**
1. 假设是否成立？
2. 不成立 → 报告"方向不可行"
3. 成立 → 分析根因并提出修复

**绝不未经假设验证就链式尝试多种修复。**

---

## Rule 6: "有意思" ≠ "有价值"

- 大规模实现前回答：**是否产生可量化的工程收益？**
- 不确定时 → 最小原型或信封计算先行
- 避免为技术炫技而优化

---

## Rule 7: 先淘汰再深耕

**多个候选方向时的正确做法：**

| 错误做法 | 正确做法 |
|---------|---------|
| 顺序实现并测试所有方向 | 信封计算淘汰明显不可行者 |
| 平均投入精力 | 最小原型验证幸存者 |
| 全量实现后再评估 | 仅对验证通过者全量实现 |

**流程：** 信封计算 → 淘汰 → 最小原型 → 全量实现（仅验证通过者）

---

## Rule 8: 及时止损

**出现以下任一情况，必须主动报告并建议停止：**

1. 信封计算显示净收益为负
2. 连续两种方法测得负收益
3. 核心假设被测量证伪
4. 发现架构阻碍（如GPU结果不被CPU消费）

**绝不默默转向新方向。**

---

## Rule 9: 切换前提交

- 每个工作里程碑必须 `git commit`
- 任何破坏性变更或方向切换前必须提交
- 使用 `git add` + `git status` 检查，避免提交无关文件

---

## 快速参考：停车信号

**出现以下信号，立即停止并重新评估：**

- [ ] 数据来源与假设不符
- [ ] 数据时序与假设不符
- [ ] 发现临时buffer/中间层存在无法绕过的原因
- [ ] 调试时间超过预期2倍
- [ ] 需要修改架构才能继续优化

---

## 快速参考：最小验证集

| 验证项 | 方法 | 通过标准 |
|-------|------|---------|
| 数据来源 | gdb打印/代码走读 | 确认数据内容和位置 |
| 数据时序 | 插入打印/断点 | 确认数据在需要时已可用 |
| 架构假设 | 画数据流图 | 确认无隐藏中间层 |
| 收益估算 | `perf` + 信封计算 | Net gain >= 5% |
| 正确性 | 最小测试用例 | CRC/MD5匹配 |

---

**记住：验证假设的成本远低于实现后发现的成本。**
